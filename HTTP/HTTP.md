# HTTP의 정의
 - HyperText Transfer Protocol 또는 HyperTexT Protocol의 약자
 - 웹에서 클라이언트(브라우저)와 서버가 서로 데이터를 주고받기 위해 사용하는 통신 규칙(프로토콜)을 의미

# 역할
 - 클라이언트와 서버를 연결을 가능하게 함


 - 예를 들면 크롬 같은 브라우저로 유튜브 같은 사이트에 접속하면<br> 브라우저가 서버에게 페이지 요청을 보내고<br> 서버가 응답을 보내는데 그 때 사용하는 약속이 HTTP이다.
 

# HTTP Method
 - 클라이언트(브라우저)가 서버에게 무슨 행동을 할지 알려주는 방식이다.
 - 대표적인 HTTP Method로는 GET, POST, PUT, PATCH, DELETE가 있다.
   - GET은 데이터를 가져올 때 사용한다.
   - POST는 데이터를 서버에 보낼 때 사용한다.
   - PUT은 데이터 전체를 수정할 때 사용한다.
   - PATCH는 데이터 일부를 수정할 때 사용한다.
   - DELETE는 데이터를 삭제할 때 사용한다.

# HTTP Status Code
 - 서버가 클라이언트(브라우저)의 요청을 처리한 뒤 결과 상태를 숫자로 알려주는 코드이다.
   - 200번대는 성공을 의미
     - ex) | 200 OK = 요청 성공 | 201 Created = 데이터 생성 성공
   - 300번대는 (리다이렉션)다른 주소로 가라는 것을 알려주는 것
     - ex) | 301 = 영구적으로 주소가 바뀐 것 | 302 Found = 주소가 잠시 이동한 것
   - 400번대는 클라이언트 오류이다
     - ex) | 400 Bad Request = 잘못된 요청 | 401 Unauthorized = 인증 필요 | 403 orbidden = 접근 금지 | 404 = Not Found 요청한 페이지 없음
   - 500번대는 서버 오류이다
     - ex) 500 Internal Server Error = 서버 내부 오류 | 502 Bad Gateway = 게이트웨이 오류 | 503 Service Unavailable = 서버 사용 불가
# HTTP의 무상태성  
 - 서버가 이전 요청을 기억하지 않는 성질이다.
 - HTTP는 기본적으로 요청 하나 처리하고 끝난다. 즉, 다음 요청은 새로운 요청으로 취급한다.
 - 예를 들면 사이트에 내가 로그인했는데 다른 요청을 하면 서버는 이 사람이 나인지 모른다는 것이다.


# HTTP의 클라이언트 - 서버 구조
 - HTTP는 클라이언트가 요청 <-> 서버가 응답 해주는 요청 <-> 응답 구조이다.
    ````
    요청 메시지 구조
     시작 줄 : 메서드(GET, POST 등), 대상 경로(/index.html), HTTP 버전
     헤더 : 데이터에 대한 정보
     헤더와 바디 구분하는 공백 줄
     바디 : 보낼 실제 데이터

    응답 메시지 구조
      상태 줄 : HTTP버전, 상태코드, 상태 메시지
      헤더 : 응답에 대한 정보
      헤더 바디 구분 공백 줄
      바디 : 서버가 보내준 데이터

 - Header의 역할
   - 데이터의 형태(Content-Type:JSON,HTML)
   - 데이터의 크기(Content-Length:)
   - 브라우저 종류(User-Agent)
   - 인증 정보(Authorization)
 
 - Body의 역할
   - 요청 시: 로그인할 때 입력한 아이디/비밀번호, 게시글 내용
   - 응답 시: 화면에 보여줄 HTML 코드, 이미지 파일, API 데이터(JSON)
