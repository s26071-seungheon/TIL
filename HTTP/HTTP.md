# HTTP의 정의
 ### HyperText Transfer Protocol 또는 HyperText Protocol의 약자
 ### 웹에서 클라이언트(브라우저)와 서버가 서로 데이터를 주고받기 위해 사용하는 통신 규칙(프로토콜) 프로토콜을 의미
 ### 예를 들면 크롬 같은 브라우저로 유튜브 같은 사이트에 접속하면<br> 브라우저가 서버에게 페이지 요청을 보내고<br> 서버가 응답을 보내는데 그 때 사용하는 약속이 HTTP이다.
 
# HTTP Method
 ### 클라이언트(브라우저)가 서버에게 무슨 행동을 할지 알려주는 방식이다.
 ### 대표적인 HTTP Method로는 GET, POST, PUT, PATCH, DELETE가 있다.
  #### GET은 데이터를 가져올 때 사용한다.
  #### POST는 데이터를 서버에 보낼 대 사용한다.
  #### PUT은 데이터 전체를 수정할 때 사용한다.
  #### PATCH는 데이터 일부를 수정할 때 사용한다.
  #### DELETE는 데이터를 삭제할 때 사용한다.

# HTTP Status Code
 ### 서버가 클라이언트(브라우저)의 요청을 처리한 뒤 결과 상태를 숫자로 알려주는 코드이다.
 ### 200번대는 성공을 의미 ex) | 200 OK = 요청 성공 | 201 Created = 데이터 생성 성공
 ### 300번대는 (리다이엑션)다른 주소로 가라는 것을 알려주는 것 ex) | 301 = 주소가 바뀐 것 | 302 Found = 다른곳으로 임시 이동
 ### 400번대는 클라이언트 오류 ex) | 400 Bad Request = 잘못된 요청 | 401 Unauthorized = 인증 필요 | 403 orbidden = 접근 금지 | 404 Not Found 요청한 페이지 없음
# HTTP의 무상태성
 ### 서버가 이전 요청을 기억하지 않는 성질이다.
 ### HTTP는 기본적으로 요청 하나 처리하고 끝난다. 즉, 다음 요청은 새로운 요청으로 취급한다.
 ### 예를 들면 사이트에 내가 로그인했는데 다른 요청을 하면 서버는 이 사람이 나인지 모른다는 것이다.
