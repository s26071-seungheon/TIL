# Controller
 - Controller는 화면(View)과 비즈니스 로직(Service)을 연결
 - 웹 요청을 받아 처리, 응답 돌려주는 역할 클래스
    - ex) aaa로 가줘 요청 -> 주소 받아들여 어디로갈지 연결

## Controller의 특징
- 요청과 응답처리
   - HTTP 요청을 처리
- URL 매핑
   - 여러가지 어노테이션으로 요청 URL과 메서드 연결
- 어노테이션 기반

 ## Controller를 쓰는 이유
 - URL마다 기능을 분리해서 관리 할 수 있다.
 - 역할이 분리되어 코드 구조가 깔끔해진다.
 - 요청 처리 집중화
   - 모든 HTTP요청 처리를 Controller에서 수행
   - 요청 처리 로직이 분산되지 않아 유지보수 용이
 - 계층 구조 유지
