# Repository
- 데이터베이스 테이블에 접근하는 메서드들을 사용하기 위한 인터페이스
- CRUD를 어떻게 처리할지 정의하는 계층
  - CRUD: Create, Read, Update, Delete

## Repository의 특징
- DB 접근(저장/조회/삭제/수정)만 담당
    - 오로지 데이터 작업만 담당
- 인터페이스만 선언하면 구현은 자동 생성된다.
    -복잡한 SQL코드 거의 없음, 기본적인 CRUD기능 모두 자동 제공
- 다양한 쿼리 자동 생성 된다.
- Entity를 기준으로 동작한다.

## Respository 써야하는 이유
- DB 접근을 더 간단하게 한다
- 코드가 더 유지보수에 좋다
- Service에서 필요한 DB 기능만 호출 가능하다
- SQL을 자동 생성 해주기 때문에 실수 확률이 줄어든다.
