# Entity
 - Java Persistence API(JPA)에서 데이터베이스를 객체 지향적으로 표현하는 클래스
   - JPA : 자바에서 DB를 객체처럼 다루도록 해주는 기술
 - 데이터베이스의 스키마를 코드로 매핑 -> 데이터베이스와 어플리케이션 간의 데이터 전송 원활하게 만들어줌
   - 스키마 : 데이터베이스 안에 데이터 구조를 정의한 설계도 / 틀

 ## Entity와 데이터베이스 테이블과의 관계
 - 하나의 Entity 클래스는 데이터베이스의 한 테이블과 매핑된다.
 - Entity 클래스의 각 변수, 필드는 테이블의 열에 해당한다.
 - 데이터베이스의 행은 Entity 클래스의 인스턴스 객체로써 표현된다.

 ## Entity를 쓰는 이유
 - DB 데이터를 객체로 다루기위해
 - JPA가 DB와 매핑 할 수 있도록 하기 위해
 - 계층 구조 유지를 위해
 - 재사용과 유지보수가 용이
