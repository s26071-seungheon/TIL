# 디자인
 ## 역할
 - 디자인은 사용자가 서비스를 쉽고 편리하게 이용할 수 있도록, 전체적인 스타일을 설계하는 역할

 ## 하는 일
 - UI설계
   - 버튼, 색상, 폰트,전체적인 분위기, 아이콘등 눈에 보이는 화면
 - UX설계
   - 사용자 입장에서 사용이 편하고 헷갈리지 않는지 확인
 - 와이어프레임 설계
   - 와이어프레임 : 디자인하기 전에 만드는 화면 설계도

 ## 사용하는 도구
 - Figma : UI/UX 여러가지 눈에 보이는 것들 디자인하는 소프트웨어
 - Adobe Photoshop : 이미지 편집 및 그래픽 디자인 소프트웨어
 - Adobe Illustrator : 벡터로 로고, 아이콘등 만드는 소프트웨어

 ## 사용하는 언어
 - HTML
 - CSS
 - JavaScript

 ## 프론트엔드와 디자인의 차이
 - 디자인이 어떻게 만들지 구상한다면 프론트엔드는 실제로 구현

 ## 프론트엔드와 디자인의 협업방법
 - 디자인 전달 도구로 같이 협업
   - 디자이너가 폰트 크기, 버튼 색상을 Figma에 배치 -> 프론트엔드는 CSS코드를 확인 가능
 - 일일이 모든 화면을 새로 그리지 않고 서로 미리 약속 정함 ex) 폰트크기, 컬러 있음
