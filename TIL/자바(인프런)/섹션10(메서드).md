# 메소드

 ## 함수(function)
  ### 함수에 값을 입력하면, 함수가 가진 연산을 처리한 다음 결과를 출력한다.
  ### 여러 번 같은 계산을 해야한다면 함수를 넣어 필요한 입력값만 넣어 함수를 호출하면 된다.
  ### 함수의 핵심은 재사용성이다.

  ## 함수 정의
   ### add(a,b) = a+b
   #### 이름이 add이고 a,b라는 두 값을 받고 a+b를 수행한다.
   ### avg(a,b) = a+b/2
   #### 이름이 avg이고 a,b라는 두 값을 받고 a+b/2를 수행한다.

## 메서드 사용
 ### 자바에서 함수를 메서드(Method)라 한다.
 ### 메서드도 함수의 한 종류라고 생각하면 된다.

## 메서드 정의
    public static int add(int a,int b) {
        System.out.println(a + "+" + b + " 연산수행");
        int sum = a+b;
        return sum;
    }
 ### 이 부분이 메서드이다. 메서드를 정의한다고 표현한다.
 ### 정의해놓고 쓰고싶을 때 불러서 쓴다.

## 메서드 선언
 ### public static int add(int a, int b)
 ### 이 부분이 메서드 선언 부분이다. 메서드 이름, 반환 타입, 매개변수 목록을 포함한다.
 ### 메서드 선언 정보를 통해 다른 곳에서 해당 메서드를 호출 할 수 있다.

## public static
 ### public : 다른 클래스에서 호출 할 수 있는 메서드라는 뜻
 ### static : 객체를 생성하지 않고 호출 할 수 있는 정적 메서드라는 뜻

## int add(int a, int b)
 ### int: 반환 타입을 정의한다. 메서드의 실행 결과를<br> 반환할 때 사용할 반환 타입을 지정한다.
 ### add: 메서드에 이름 부여한다. 이 이름으로 메서드를 호출한다.
 ### (int a, int b): 메서드를 호출할 때 전달하는 입력 값을 정의한다.

## 메서드 본문
        System.out.println(a + "+" + b + " 연산수행");
        int sum = a+b;
        return sum;
 ### 메서드가 수행해야하는 코드 블록이다.
 ### 메서드를 호출하면 메서드 본문이 순서대로 실행된다.
 ### 메서드 실행결과를 반환할려면 return문을 써야한다.

## 메서드 호출
 ### int sum = add(5,10); 같이 이름에 입력 값을 전달하면 된다.
 ### 메서드 호출이 끝나면 메서드 정의에 사용한 매개변수는 정의 한 값이 모두 사라진다.

## 용어정리
 ### 인수: 메인에서 메서드로 넘기는 값을 인수라 한다.
 ### 매개변수: 메서드를 정의 할 때 선언한 변수<br> 메서드를 호출 할 때 인수를 넘기면 매개변수에 대입된다.
 
## 반환 타입이 없을 때
 ### 반환 타입에 void를 친다.
 ### return 값이 필요없다.

## 반환 타입이 있을 때
 ### return 값이 꼭 필요하다.
 ### 인수와 매개변수가 필요하다.

# 메서드 대원칙1
 ### 자바는 항상 변수의 값을 복사해서 대입한다.

# 메서드 오버로딩
 ### 자바는 이름뿐만 아니라 매개변수로도 구분이 가능하다.
 ### 따라서 이름이 같고 매개변수가 다른 메서드 정의가 가능하다.
 ### 매개변수만 다른 것을 메서드 오버로딩이라고 한다.
 
## 오버로딩 규칙
 ### 매개변수의 타입 및 순서가 다르면 오버로딩을 할 수 있다.
 ### 반환 타입은 인정하지 않는다.

## 메서드 시그니처
 ### 메서드 이름 + 매개변수 타입(순서)
 ### 메서드를 구분할 수 이는 기준이라고 이해
 ### 메서드 시그니처가 다르면 다른 메서드로 이해
