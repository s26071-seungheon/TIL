# 객체지향의 정의
 ### 현실 세계의 사물(객체)을 기준으로 프로그램을 설계하는 방식
 ### 데이터(속성)와 기능(메서드)을 하나의 객체로 묶어서<br> 프로그램을 구성하는 프로그래밍 패러다임

# 캡슐화
 ### 데이터(변수)와 기능(메서드)을 하나로 묶고, 외부접근을 제한하는 것

 ## 필요한 이유
 ### 데이터를 함부로 바꾸면 버그 발생한다
 ### 객체 내부 상태 안전하게 보호한다

# 상속
 ### 기존 클래스의 속성과 기능을 물려받는 것

 ## 필요한 이유
 ### 코드 재사용이 된다
 ### 공통 기능 관리가 쉽다

# 다형성
 ### 같은 메서드가 상황에 따라 다르게 동작
 
 ## 필요한 이유
 ### 코드 유연성이 증가한다
 ### 유지보수가 쉽다

# 추상화
 ### 불필요한 부분을 생략하고 객체의 속성 중<br>가장 중요한 것에만 중점을 두어 개략화 하는 것

 ## 필요한 이유
 ### 복잡도가 감소한다.
 ### 사용자는 사용법만 알면된다.
